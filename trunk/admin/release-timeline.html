Shipped — from the real release notes
v1.0.0 Initial on-device inference release — the rebrand from PocketCoder to Trunk, first version published to both repos.
v1.1.0 Developer Options (diagnostics suite, failure log, storage inspector), plain-text file attachments, live inference-unit indicator. Fixed Flow canvas bugs, markdown gaps, models echoing their own system prompt, and app-wide em-dash rendering.
v1.2.0 GPU (Adreno/OpenCL) acceleration — detects and offloads inference to Qualcomm Adreno 700+ GPUs, with a Settings toggle and onboarding step.
v1.3.0 Hexagon NPU acceleration on supported Qualcomm chipsets.
v1.3.1 Chat sessions stay pinned to the inference unit (NPU/GPU/CPU) they started on, even if the Settings toggle changes mid-chat. Composer token meter no longer dips after sending.
v1.3.2 Flows got their own generation settings (temperature/topP/topK/context/max tokens). "Run via Flow" inside a Project now correctly uses the project's settings. Added android:largeHeap to stop background low-memory kills.
v1.3.3 Fixed the keyboard covering form fields in New Project, New Flow, and Agent Editor modals.
v1.3.4 Developer Options promoted to a mainstream Settings section. Opt-in Usage Logging with explicit consent + Send Logs. App version and release notes on the About screen. Keep Screen On setting. Several Flow/download reliability fixes.
v1.3.5 Hotfixv1.3.4 crashed on launch for every user (NoClassDefFoundError from an Expo SDK version mismatch in expo-keep-awake). Also removed the backend URL debug display.
v1.3.6 Pre-release — pending device verification

Recommended models, NPU/GPU fixes, crash-report hardening

  • HF search "Recommended" list — filters to models sized for the device's RAM by default; a disclaimer flags non-quantized or non-chat models before download
  • Fixed stale NPU/GPU label on Inference; toggle now hides (with an info button) on unsupported devices
  • Crash reports now require a shared secret + rate limit, and attach recent error-log entries
  • Onboarding warnings require a 5-second read before continuing
Tagged and built on the draft repo, marked pre-release on GitHub pending a real device test. Not yet promoted to the public trunk-android repo.
v1.4.0a Next up

Speech-to-text and text-to-speech

  • Audio input — speech-to-text for the composer, in place of (or alongside) typing
  • Audio output — text-to-speech for model responses
Versioning rule as given: if this turns out easy and doesn't need heavy testing, it ships directly as v1.4.0. If it needs real testing first, it ships as pre-release v1.4.0a — full v1.4.0 follows once that testing is done.
v1.3.7 or v1.4.1 Depends on v1.4.0 path

Model storage moves to downloads/

  • Downloaded GGUF models saved to the public Downloads folder instead of internal app storage — so they survive an APK uninstall/reinstall
  • Storage permission requested at app startup; if denied, re-prompt every time the user opens the Models or Hugging Face page
Versioning rule as given: ships as v1.3.7 if v1.4.0 goes out as the pre-release v1.4.0a. Ships as v1.4.1 if v1.4.0 ships as a proper full release instead.
v1.5 Speculative

External API support + agent marketplace

  • Support for any external LLM API, not one fixed provider
  • Agents can run against those external APIs, not just on-device models
  • Prebuilt agent packages, designed in-house and hosted on GitHub, downloadable from inside the app
  • Open the same mechanism to the community — users design and submit their own agents
  • Explore compatibility with Google's ADK (Agent Development Kit) — floated as "just a thought," not committed
v1.6 Speculative

Phone as local host

  • A phone can act as a host, serving its loaded model to other devices on the same network
  • Android-to-Android: host mode to send, client mode to receive
  • Android-to-PC: the PC side connects via terminal — something SSH-like to ping the host and pull data
v1.7 Speculative

Multimodal input

  • Input beyond plain text — images and other modalities feeding into a chat or Flow
v1.8 Speculative

Embeddings + RAG

  • On-device embeddings and retrieval-augmented generation over the user's own documents/models
v1.9 Speculative

Acyclic graph agents

  • Tree-structured agent flows — branching beyond today's linear Playground Flows
v1.10 Speculative

Cyclic graph agents

  • Loops in the agent flow graph — agents that can revisit earlier steps
Unresolved — flagged alongside this roadmap

Who is this app actually for, and why would someone switch from Gemini, Claude, or ChatGPT to use it?

Every line above adds capability, but none of them is yet the answer to that question. Worth revisiting before v1.5 — the two ideas on this board with the most "nobody else does this" shape are v1.5's community agent marketplace and v1.6's phone-as-host.
